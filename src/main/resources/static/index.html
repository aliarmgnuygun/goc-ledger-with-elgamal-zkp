Manual Transfer — free experiment

Three accounts (Alice=20, Bob=10, Carol=5) start with encrypted balances. Pick a sender, a receiver and an amount, then try an honest transfer or an attack.

Note: the ledger flow works only with DL and EC. Bulletproofs is not here because a transfer needs, in addition to a range proof, a ciphertext-equivalence proof (new balance = current − amount); the Bulletproofs library we use only gives a standalone range proof and cannot produce an equality proof inside the same ElGamal scheme. The speed/size comparison of the three schemes is done in the benchmark charts.

Warning: a larger bit length makes a transfer slower. EC stays fast at 32-bit (~0.3 s), but DL at 32-bit can take ~20 s per transfer (2048-bit modPow × 32 bits). If you want to try 32-bit, we recommend EC; this is the live version of the "DL is the slowest" finding in the charts.
